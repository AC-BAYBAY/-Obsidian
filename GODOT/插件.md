# Godot 插件整理

> 来源：B站视频——16款优秀且免费的 Godot 游戏引擎插件
> 整理日期：2026-05-14

---

## 一、动画与图像

### 1. Aseprite to Godot (ASTOP)
- **功能**：将 `AnimatedSprite2D` 中的所有动画一键转换为 `AnimationPlayer` 动画轨道。
- **解决问题**：Godot 原生没有提供从 `AnimatedSprite2D` 到 `AnimationPlayer` 的转换功能，做好的动画需要手动一个个复制过去，非常麻烦。ASTOP 一键解决这个痛点，转换后即可方便地添加帧事件或 K 帧。
- **适用场景**：人物或技能动画比较多的游戏。

### 2. Inputality
- **功能**：多格式图像与动画导入工具包。支持从 ASEPRITE、Crea、Pencil、Piskel、Pixel Ram 等常用帧动画软件导入动画资产，自动生成对应的资源和节点结构。
- **解决问题**：帧动画与 Godot 配合的正常流程（导出→导入→切帧→设置）次数多了会非常繁琐。Inputality 无需手动切图，修改动画后 Godot 自动同步，节约大量重复劳动。
- **适用场景**：使用外部帧动画软件制作资产的 Godot 项目。

### 3. Spine 插件（官方）
- **功能**：Spine 官方为 Godot 量身打造的新版插件，以 GDExtension 方式融入引擎，已完成对 Godot 4.5.1 和 3.6.2 的适配。
- **解决问题**：长期以来 Godot 引擎对 Spine 动画的支持不足。Spine 官方的适配使插件与 Godot 原生动画、物理和渲染系统协作更加紧密。
- **适用场景**：追求优秀 Spine 骨骼动画效果的开发者。

## 二、相机与场景

### 4. Phantom Camera
- **功能**：为 2D 和 3D 项目提供智能相机行为，包括平滑跟随目标、动态框定多个物体、实现第三人称视角、不同机位间流畅切换等。灵感来源于 Unity 的 Cinemachine。
- **解决问题**：镜头表现直接影响游戏质感，自建复杂相机系统工作量大。Phantom Camera 与编辑深度集成，通过参数面板即可实时预览画面和死区范围。
- **适用场景**：需要高质量镜头控制的 2D/3D 游戏项目。

### 5. Scene Manager
- **功能**：场景管理和过渡控制插件，支持淡入淡出和多种高度自定义的转场效果。
- **解决问题**：游戏场景切换看似简单，但想要把场景切换时的过渡控制做好，不依赖插件需要花不少心思。原生切换效果生硬。
- **适用场景**：中小项目快速管理场景、复杂项目的场景调度需求。

## 三、代码与编辑体验

### 6. Script IDE
- **功能**：强化 Godot 内置脚本编辑体验，提供更清晰的代码结构视图、函数与变量快速跳转、搜索增强、更友好的编辑界面布局。
- **解决问题**：项目逐渐变大时，原生脚本管理效率低。适合习惯了 VS Code 布局又想在内置代码编译器里写代码的开发者。
- **适用场景**：项目规模增长后的代码组织与浏览效率提升。

### 7. To Do Manager
- **功能**：自动收集代码中写下的 `TODO` 或 `FIXME` 标签，集中在编辑器底部的 Tag 页面显示，形成任务清单。
- **解决问题**：开发过程中总有一些要「以后再优化」的地方，容易遗忘。To Do Manager 让你不会忘记以后要改进的代码位置。
- **适用场景**：任何 Godot 游戏项目。

## 四、游戏逻辑与系统

### 8. Dialogue Manager
- **功能**：用于构建游戏对话系统的插件，支持变量控制、条件判断和分支选择。支持 BBCode 和多语言本地化功能。
- **解决问题**：可以像写代码一样编写对话和剧情，轻松实现剧情分流和角色互动逻辑。
- **适用场景**：RPG 剧情向游戏、有多个分支对话的项目。

### 9. Godot State Charts
- **功能**：有限状态机插件，支持使用节点加纯代码方式控制 NPC 和 BOSS 各阶段行为，有独立界面可实时观察行为逻辑流向。
- **解决问题**：有限状态机常用于游戏中的 NPC 和 BOSS 战斗逻辑，State Charts 相比其他状态机插件多了可视化观测能力。
- **适用场景**：多阶段复杂 BOSS 战斗的游戏项目。

### 10. Sound Manager
- **功能**：统一管理音乐和音效的播放，支持 0~1 的音量控制和切换逻辑。可将游戏音频分为游戏音乐、环境音乐、游戏音效、UI 音效四个通道独立控制，统一输出到 Master 主通道。
- **解决问题**：游戏中音乐和音效多了之后单独管理容易混乱，且原生 Godot 音频 API 以分贝为单位不方便。Sound Manager 让声音变成一个系统。
- **适用场景**：对音频控制要求未达到需要 FMOD/Wwise 中间件程度的项目。

### 11. Godot SFXR
- **功能**：在引擎内部生成随机音效（跳跃声、攻击声等），无需额外寻找资源。可像普通音频文件一样使用，也可像在原始 SFXR 软件中一样编辑。
- **解决问题**：制作游戏原型时经常需要简单音效，专门找资源费时。
- **适用场景**：快速开发原型、Game Jam 项目。

### 12. CSV Data Importer
- **功能**：读取 CSV 表格数据，修改表格中数据后游戏对应数值自动同步更新。
- **解决问题**：数值驱动的游戏如果把数值写死在代码里，后期调整会非常痛苦。该插件将数值统一放在类似 Excel 的表格中管理，修改后自动生效，无需反复修改代码。
- **适用场景**：频繁调整数值的开发阶段。

## 五、输入与编辑效率

### 13. Input Helper
- **功能**：专业的设备输入管理插件，统一处理键盘、手柄和其他输入设备，简化动作映射和按键识别，支持按键重绑定和多种控制器。
- **解决问题**：使用原生 Godot 内置 API 实现玩家自定义手柄和键盘按键映射非常麻烦。Input Helper 让跨设备输入支持的实现变得简单。
- **适用场景**：需要手柄+键盘跨设备输入支持、允许玩家自定义按键映射的游戏。

### 14. Blender 3D Shortcut
- **功能**：让 Godot 的 3D 编辑操作更接近 Blender 习惯。支持 G/R/S 快捷键进行移动/旋转/缩放，XYZ 轴约束、Shift 键精细控制、Alt 键快速还原。
- **解决问题**：熟悉 Blender 的开发者切换到 Godot 时操作习惯不同。该插件将 Blender 常用快捷键映射过来，学习成本几乎为零。
- **适用场景**：熟悉 Blender 的 3D 游戏开发者。

## 六、优化与发布

### 15. Godot Size
- **功能**：分析游戏项目体积占用情况的插件，扫描整个 Godot 项目文件夹，按文件尺寸从大到小排列。
- **解决问题**：游戏优化打磨阶段，体积控制变得重要。该插件帮助快速定位体积异常的资源文件。
- **适用场景**：移动端或 Web 项目的体积优化阶段。

### 16. Godot Steam
- **功能**：为 Godot 集成 Steamworks API 的开源工具，支持 Windows、Mac OS 和 Linux 平台。封装了成就系统、排行榜、玩家统计、云存档等常用接口。
- **解决问题**：让 Godot 游戏接入 Steam 平台功能时无需直接对接 V 社底层 C++ SDK，开箱即用。但要注意 Godot 版本、Godot Steam 版本和 Steamworks 版本最好一一对应，避免 Bug。
- **适用场景**：计划发布到 Steam 平台的 Godot 游戏。

- 在开源大佬的帮助下完美移植到了 Godot。
