# Resource 存档系统

## 核心概念

基于 `Resource` 类实现存档，适合保存多样复杂的场景数据。默认 Resource 不能保存数据，需要**自定义继承 Resource 的子类**作为数据载体。

## 关键节点/类及作用

| 节点/类               | 作用              | 关键用法                                             |
| ------------------ | --------------- | ------------------------------------------------ |
| **自定义 Resource 类** | 存档的数据结构载体       | `extends Resource` + `class_name SceneData`      |
| `@export var`      | 声明可序列化保存的变量     | `@export var player_position: Vector2`           |
| `ResourceSaver`    | 将 Resource 写入硬盘 | `ResourceSaver.save(data, "user://...")`         |
| `ResourceLoader`   | 从硬盘读取存档         | `ResourceLoader.load("user://...") as SceneData` |
| `PackedScene`      | 将整个节点打包保存       | `.pack(node)` 打包 → `.instantiate()` 还原           |
| **Group（分组）**      | 给节点打标签，批量获取     | `get_tree().get_nodes_in_group("box")`           |

## 使用方法

### 1. 创建自定义存档数据类

```gdscript
extends Resource
class_name SceneData

@export var player_position: Vector2
@export var is_facing_left: bool
@export var box_array: Array[PackedScene]
```

### 2. 保存存档

```gdscript
func save():
    var data = SceneData.new()
    data.player_position = player.global_position
    data.is_facing_left = player.get_child(0).flip_h

    # 保存场景中的节点（打包成 PackedScene）
    var boxes = get_tree().get_nodes_in_group("box")
    for box in boxes:
        var box_packed = PackedScene.new()
        box_packed.pack(box)
        data.box_array.append(box_packed)

    ResourceSaver.save(data, "user://save_data.tres")
    print("保存成功")
```

### 3. 读取存档

```gdscript
func load_game():
    var data = ResourceLoader.load("user://save_data.tres") as SceneData
    if not data:
        return

    # 恢复玩家
    player.global_position = data.player_position
    player.get_child(0).flip_h = data.is_facing_left

    # 删除旧箱子，生成新箱子
    for box in get_tree().get_nodes_in_group("box"):
        box.queue_free()

    for box_packed in data.box_array:
        var box_node = box_packed.instantiate()
        get_tree().current_scene.add_child(box_node)
```

## 注意事项

### 文件路径与格式

- **`user://` 路径**：不在游戏文件夹内，更新/删除游戏不影响存档。不同系统路径不同。
- **`.res` 后缀**：二进制格式，乱码但性能好、更安全。
- **`.tres` 后缀**：文本格式，人类可读，便于开发和调试。

### `@export` 关键字

必须加 `@export`，变量才能被序列化保存到文件中。

### 大量物理物体加载 Bug

- **问题**：大量 `RigidBody2D` 同时生成时，物理引擎计算跟不上，物体会**乱飞**。
- **解决方案**（[!] 重点）：
  1. 生成时设置 `box_node.freeze = true`（暂停物理计算）
  2. 所有箱子生成完毕后，等待短暂时间（0.1s）：
     ```gdscript
     await get_tree().create_timer(0.1).timeout
     ```
  3. 再遍历解冻所有箱子：
     ```gdscript
     for box in get_tree().get_nodes_in_group("box"):
         box.freeze = false
     ```

### 与其他存档方式对比

| 方式 | 特点 |
|-----|------|
| **Resource** | 较均衡，适合复杂场景数据 |
| ConfigFile | 适合简单配置 |
| JSON / 二进制序列化 | 各有侧重（大小/安全/灵活） |

## 场景节点连接

- 在 UI 节点挂载存档脚本
- 用 `@export var player: Node2D` 拖拽绑定玩家节点
- 保存/读取按钮的 `pressed` 信号连接到脚本的 `save()` / `load_game()` 函数
