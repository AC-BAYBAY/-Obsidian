# 入门节点说明

Godot 节点体系的核心概念与常用节点详解。

---

## 01 Node（基础节点）

- **Node** 是 Godot 最基础的构建模块，节点之间形成树状结构。
- 一个节点可以有无限个子节点，但**兄弟节点的名字必须唯一**。
- 通过继承实现常用延伸节点，分为 2D、3D、UI 三大类。
- 节点树称为**场景**，可重复使用且可独立运行。
- 可以为节点添加脚本赋予自定义能力。

### 生命周期相关虚函数

| 函数 | 触发时机 |
|------|----------|
| `_enter_tree()` | 节点进入场景树时（从上往下触发） |
| `_ready()` | 该节点的所有子节点都进入场景树后（从下往上触发） |
| `_process(delta)` | 每一帧执行 |
| `_physics_process(delta)` | 固定时间间隔执行（物理帧，用于物理计算） |
| `_exit_tree()` | 节点离开场景树时（用于重置/销毁数据和外部依赖） |

---

## 02 Node3D（三维节点）

- 所有红色节点都继承 `Node3D`，是 Godot 最基础的 3D 节点。
- 提供**可见性**属性和基础**变换**属性：位置（Position）、旋转（Rotation）、缩放（Scale）。
- 子节点继承父节点的变换。
- 可在工具栏切换**全局坐标**和**本地坐标**。
- 常用变换方法（带 `global` 的针对全局坐标，不带的针对本地坐标）：
  - `translate()`, `rotate()`, `scale()`
  - `global_translate()`, `global_rotate()` 等

---

## 03 CanvasItem（画布项目）

- 所有 2D 相关节点的基类，提供**渲染**和**绘图能力**。
- 继承自 `Node`，是抽象基类，不能直接实例化。
- 两个主要分支：`Node2D`（2D 游戏节点）和 `Control`（UI 节点）。

### 渲染顺序

- 同一 `CanvasLayer` 中，按场景树顺序渲染。
- 可通过修改 `z_index` 属性调整单个节点渲染顺序。
- 可通过创建不同 `CanvasLayer` 分离渲染模块（如背景、游戏层、UI 层）。

### 常见属性

| 属性 | 作用 |
|------|------|
| `visible` | 显示/隐藏物体 |
| `modulate` | 调整自身及子节点颜色（用于着色、淡入淡出动画） |

### 底层绘图接口

通过重写 `_draw()` 函数，使用 `draw_*` 系列方法绘制图形（如 `draw_rect`, `draw_circle`, `draw_line` 等）。

---

## 04 Node2D（二维节点）

- 继承自 `CanvasItem`，是所有 2D 游戏相关节点的基类（蓝色节点）。
- 提供基础的**变换属性**：位置（Position）、旋转（Rotation）、缩放（Scale）、倾斜（Skew）。
- 子节点继承父节点的变换，可实现复杂层级结构和动画。
- 可将节点设为顶层，继续使用全局坐标。

### 变换方法

- 本地坐标：`position`, `rotation`, `scale`
- 全局坐标：`global_position`, `global_rotation`, `global_scale`

---

## 05 Control（用户界面节点）

- 继承自 `CanvasItem`，负责 UI 渲染、响应点击/拖拽、自适应屏幕大小、应用主题。

### 基础变换属性

- 大小（Size）、位置（Position）、旋转（Rotation）、缩放（Scale）、轴心偏移（Pivot Offset）
- 旋转和缩放围绕**轴心点**进行，轴心设为 `(Size / 2)` 可沿中心变换。

### 锚点（Anchor）

- 四个锚点将 Control 的四个角"钉"到父节点上。
- 若当前节点为场景根节点，锚点作用于游戏视窗。
- 若父节点为 `Container` 类型 Control，锚点失效，大小和位置由 Container 控制。
- 内置 16 种锚点预设，分为四组：定点、定宽、定高、全自动。

### 鼠标/触摸事件

- 重写 `_gui_input(event)` 可处理输入类型（点击、移动等）。
- 通过**鼠标过滤（Mouse Filter）** 决定当前节点是否处理事件，或是否向上传递。

### 主题（Theme）

- 可在主题编辑器中统一设置 UI 风格。
- 可在项目设置中配置默认主题，通过代码切换皮肤。

---

## 06 CollisionObject（碰撞节点）

- 用于自定义碰撞范围，实现物理碰撞效果。
- 分为 2D 和 3D 版本（`CollisionObject2D` / `CollisionObject3D`），特性相同。

### 两大分支

| 类型 | 作用 |
|------|------|
| **PhysicsBody** | 会导致实际物理碰撞 |
| **Area** | 仅检测范围内的其他 CollisionObject，不产生物理碰撞 |

### 碰撞形状

- 通过添加 **`CollisionShape`**（内置预设形状）或 **`CollisionPolygon`**（自定义多边形）定义物理范围。
- 高精度碰撞形状通常由模型自动生成（例如使用 `ConcavePolygonShape` 或分析网格）。

### 物理层（Layer & Mask）

- `collision_layer`：节点所在的物理层。
- `collision_mask`：节点检测的物理层。
- 只有互相进入对方检测范围，才能正确触发碰撞/检测。
- 可在项目设置中为物理层命名（如玩家、敌人、地面、道具）。

### 鼠标/输入事件

- 对于 `CollisionObject2D` / `CollisionObject3D`，可通过信号或重写 `_input_event` 处理点击、悬浮等交互。
  - `mouse_entered` / `mouse_exited`：鼠标悬浮进入/离开
  - `input_event`：处理点击、移动等（常用于拖拽）

---

## 07 常用碰撞节点（2D 平台游戏示例）

| 节点 | 用途 |
|------|------|
| **`StaticBody2D`** | 静态平台（关卡中不动的平台） |
| **`AnimatableBody2D`** | 移动平台（配合 `AnimationPlayer` 移动）。注意：与 `StaticBody2D` 移动时的区别在于 `AnimatableBody2D` 会推动其他物体 |
| **`CharacterBody2D`** | 玩家角色（通过脚本处理输入实现移动） |
| **`RigidBody2D`** | 完全基于物理控制的物体（如足球） |
| **`Area2D`** | 感应区域（如龙门架触发胜负） |

**总结**：`StaticBody`（静态）→ `AnimatableBody`（动态移动并推动物体）→ `CharacterBody`（角色控制）→ `RigidBody`（物理模拟）→ `Area`（感应触发）。

---

## 08 RigidBody（刚体）

- 运动完全由物理引擎控制。
- 分为 2D 和 3D 版本。

### 物理属性

| 属性 | 说明 |
|------|------|
| `mass` | 质量（越大越难被推动） |
| `friction` | 摩擦力。碰撞时，两个物体的摩擦力会综合计算（通常取平均或按物理材质规则） |
| `bounce` | 弹性。若希望碰撞时吸收能量，可将 `bounce` 设为 0 并配合阻尼或自定义力 |
| `gravity_scale` | 重力系数（可为 0 或负数） |
| `center_of_mass` | 重心（可手动设置，需要将 `center_of_mass_mode` 设为 `CUSTOM`） |
| `inertia` | 惯性（施加力矩使物体旋转/停止，设为 0 表示由引擎自动计算） |

### 休眠与冻结

| 属性 | 说明 |
|------|------|
| `can_sleep` | 停止后自动进入睡眠（节省性能） |
| `lock_rotation` | 锁住旋转 |
| `freeze` | 完全冻结（被冻结的刚体不再受物理影响）。对于永远冻结的物体，更推荐使用 `StaticBody` 或 `AnimatableBody` |
| `freeze_mode` | `FREZE_MODE_STATIC`：移动时不会产生碰撞响应；`FREZE_MODE_KINEMATIC`：移动时仍会产生碰撞响应 |

### 物理计算

- `custom_integrator`：手动在 `_integrate_forces(state)` 中实现物理响应。
- **连续碰撞检测（CCD）**：预测移动路径避免漏过小/快速物体。
  - `DISABLED`：无预测
  - `CAST_RAY`：射线模式（速度中等，精度中等）
  - `CAST_SHAPE`：形状模式（最慢但最精准）

### 接触与事件

- `contact_monitor`：开启后碰撞时触发事件。
- `max_contacts_reported`：最大同时报告碰撞数量。

### 力和冲量方法

- 线性力：`apply_force`
- 线性冲量：`apply_central_impulse`
- 旋转力：`apply_torque`
- 旋转冲量：`apply_torque_impulse`

---

## 09 CharacterBody（角色运动体）

- 通过脚本移动的物理体，不受物理引擎自动控制，但会影响路径上的其他物理体（如推动 `RigidBody`）。
- 继承自 `CollisionObject`（通过 `CharacterBody2D` / `CharacterBody3D`）。
- 适用于玩家或需要精确移动控制的角色。

### 两种移动接口

| 方法 | 特点 |
|------|------|
| `move_and_collide(velocity)` | 沿速度方向移动直到碰撞，返回 `KinematicCollision` 信息，可自定义响应（如反弹） |
| `move_and_slide()` | 自动处理滑动、斜坡、平台等，并根据碰撞自动修改速度，无需手动处理每个碰撞响应 |

### move_and_slide 相关参数

| 参数 | 说明 |
|------|------|
| `velocity` | 速度向量，碰撞后自动修改 |
| `motion_mode` | `MOTION_MODE_GROUNDED`（平台游戏，区分地面/墙壁/天花板）或 `MOTION_MODE_FLOATING`（俯视视角，所有碰撞视为墙体） |
| `up_direction` | 向上的方向（用于 `GROUNDED` 模式下判断地面、墙壁、天花板） |
| `slide_on_ceiling` | 碰到天花板时是否允许继续滑行（`true`）或直接掉落（`false`） |
| `stop_on_slope` | 停止时是否在斜坡上停留（`true`）或向下滑动（`false`） |
| `constant_speed` | 斜坡上速度是否恒定（`true`）或受坡度影响（`false`） |
| `wall_min_slide_angle` | 在 `FLOATING` 模式下，墙体允许的最小滑动角度（弧度） |

### 适用场景对比

- **CharacterBody**：角色移动逻辑简单（只处理重力、输入），不依赖复杂物理交互时推荐使用。
- **RigidBody**：需要完全基于物理引擎（力、冲量、碰撞反弹等）时选用。
